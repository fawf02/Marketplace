# 📱 StudentHub - Маркетплейс для студентов

**StudentHub** - это современный маркетплейс для студентов, созданный с использованием Vue 3, Vite и Tailwind CSS. Проект демонстрирует профессиональный подход к разработке frontend-приложений с применением лучших практик UI/UX дизайна.



### Требования
- Node.js 16.x или выше
- npm 8.x или выше

### Установка и запуск

```bash
# Клонировать репозиторий
cd VueProject

# Установить зависимости
npm install

# Запустить dev-сервер
npm run dev
# Доступно на http://localhost:5174

# Собрать для production
npm run build

# Превью production-сборки
npm run preview
```

## 💡 Советы для разработчиков

1. **Используйте `card-hover` класс** для карточек с эффектом
2. **Все кнопки должны иметь иконки** с правой стороны
3. **Проверяйте адаптивность** с помощью браузерного DevTools
4. **Используйте slate цвета** вместо gray для консистентности
5. **Применяйте hover-эффекты** ко всем интерактивным элементам
6. **Используйте Vue DevTools** для отладки (Alt+Shift+D)

Этап 4: Управление состоянием и Корзина
Цель: Реализовать логику добавления товаров и их хранение между переходами по страницам.

Ключевые компоненты:
stores/cartStore.js (Pinia): Хранилище состояния корзины.
components/CartDrawer.vue или views/CartView.vue: Визуализация корзины.
Рекомендация: Используйте localStorage внутри Pinia, чтобы корзина не сбрасывалась при обновлении страницы.

Промпт для генерации: "Настрой State Management с помощью Pinia для корзины.
Создай стор useCartStore. В нем должны быть действия: addToCart(product), removeFromCart(id), clearCart().
Реализуй геттеры: totalPrice (общая сумма) и itemsCount (количество товаров).
Создай компонент CartDrawer.vue (выезжающая боковая панель), который отображает список выбранных товаров, позволяет менять их количество (+/-) и показывает итоговую сумму.
Реализуй сохранение корзины в localStorage."



Этап 5: Авторизация и Профиль
Цель: Имитация входа в систему и личный кабинет студента.

Ключевые компоненты:
views/LoginView.vue / RegisterView.vue: Формы с валидацией.
views/ProfileView.vue: Данные пользователя и история заказов.
stores/authStore.js: Хранение статуса авторизации (isAuthenticated).

Промпт для генерации: "Реализуй простую систему авторизации.
Создай стор useAuthStore, который хранит текущего пользователя и флаг isAuthenticated. Сделай методы login(email, password) и logout(). (Логин может быть фиктивным, просто проверка на непустые поля).
Создай красивую форму входа LoginView.vue с валидацией полей.
Создай страницу ProfileView.vue, которая доступна только авторизованным пользователям (настрой Navigation Guard в роутере). На странице профиля выведи фиктивную 'Историю заказов'."



Этап 6: Оформление заказа (Checkout)
Цель: Финальный шаг воронки продаж.

Ключевые компоненты:
views/CheckoutView.vue: Форма адреса/доставки и выбор оплаты.
Модальное окно "Спасибо за заказ".

Промпт для генерации: "Создай страницу оформления заказа CheckoutView.vue.
Раздели страницу на две колонки: слева форма данных (Имя, Общежитие/Адрес, Телефон), справа — саммари заказа (список товаров и итоговая сумма из стора корзины).
При нажатии 'Оформить заказ' покажи красивое модальное окно с сообщением 'Заказ успешно создан!' и очисти корзину через Pinia."
